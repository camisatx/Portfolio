About

I am passionate about building disruptive technologies to help humanity as a whole. I am a programmer first, but also enjoy web design, system architectures, computer hardware and data structures. I believe that technology can be used to solve (or at least aid) many of the difficult problems we face today and in the future.

As a autodidact of programming, I focus on the real world applications of the techniques I know. I believe heavily in the importance of systematic best practices for all code, thus I follow PEP8 standards (Python), heavily rely on Git as my VCS and follow the TDD process when developing code (my unit test just confirmed this ;) ).

I would rather slowly build solid code than quickly make messy code. Ultimately, I hate shitty code. Code is art, so make it beautiful!
